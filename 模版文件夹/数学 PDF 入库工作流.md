# 数学 PDF 入库笔记模板与插件工作流

## 目标

后续上传数学 PDF 后，只参考当前仓库的数学体系，不使用 408 模板。目标是把 PDF 内容整理成 Obsidian 可复习、可检索、可链接的数学笔记：章节基础知识、公式卡、方法卡、思维导图入口、题目 / 错题卡、同类题目汇总、二刷复盘和图解资料。

> [!warning] 边界
> 本工作流只指导数学笔记入库；不默认修改 `.obsidian/`、插件目录、主题配置、PDF/PPT 原始资料或已有手写补充。

## 笔记类型

- `基础知识`：章节知识点主体，适合定义、定理、公式、性质、例题、易错点。
- `公式卡`：单个公式、定理或判别准则的条件、变量、适用范围、反例。
- `方法`：抽象出通用解题套路，把“某一题的技巧”升级为“同类题入口”。
- `思维导图`：每章入口页，通常命名为 `x-0 xxx思维导图.md`，用标题层级配合 Mind Map NextGen。
- `题目` / `错题`：题干、题图、解法、错因、关键一步、二刷记录。
- `同类题目汇总`：用 Dataview 按标签、类型、评分、掌握状态动态汇总。
- `章节复习仪表盘`：按章节汇总基础、公式、方法、题目和待确认事项。
- `图解`：保存 Excalidraw、函数图像、几何图形、积分区域、矩阵结构图等可视化材料。
- `二刷复盘`：专门记录错题二刷 / 三刷后的变化，避免把复盘塞回原题造成混乱。
- `补充知识`：PDF 中有价值但不适合放进主线章节的旁支理解、证明或扩展。

## PDF 入库步骤

1. 识别 PDF 属于高数、线代、概率论的哪一章；原始 PDF 不移动、不删除。
2. 确认 PDF 页数、来源路径、是否扫描型 PDF；扫描型资料先 OCR。
3. 提取目录、页码范围、知识点、例题、图表；OCR 不清处标记 `待确认`。
4. 先建立或补全 `基础知识` 与 `公式卡`，保持公式、条件、结论完整，并把概念和做题动作分开。
5. 再抽出可复用的 `方法`，写清识别信号、操作步骤、适用边界、反例和错题回流入口。
6. 对有价值的题目生成 `错题` 或 `题目` 卡；题图放入当前章节 `fig` 目录或 Obsidian 附件目录。
7. 遇到图像依赖强的内容，新建 `图解` 卡并链接 Excalidraw 或截图。
8. 更新本章 `思维导图` 和 `章节复习仪表盘`，把新知识、方法、题型、例题索引、错题回流和后续章节链接接入已有网络。
9. 如某类题目数量足够，新增或更新 `同类题目汇总`，用 Dataview 自动拉取。
10. 最后检查 frontmatter、链接、图片引用、公式渲染、Dataview 查询、命题信号表、例题索引、错题回流和 `待确认` 任务。

## 复习导向章节结构

章节入口页和较大的章节整理稿优先采用以下结构：

1. `考点权重总览`：按 `基础`、`高频`、`易混`、`后续依赖` 标注复习动作。
2. `知识体系`：只放主干层级和关键链接。
3. `概念区`：定义、公式、定理、条件、变量解释。
4. `做题动作区`：看到题目后怎么判断、怎么下手、怎么避免误用。
5. `命题信号 -> 方法入口`：用表格连接题干特征、方法、关联笔记和易错提醒。
6. `典型例题索引表`：只列编号、考点、方法入口、是否建议拆卡、复核状态；题解和答案必须回 PDF 复核。
7. `分主题易错点`：按知识模块拆分，不把易错点堆成一段。
8. `反例 / 边界情况`：记录不能套用的条件和反例。
9. `Dataview 查询块`：汇总基础、方法、题目 / 错题、AI 草稿待复核。
10. `后续章节链接` 与 `错题回流入口`：把本章知识接到后续章节和错题复盘。
11. `最小记忆清单` 与 `待确认`：前者用于快速复盘，后者必须写成可执行复核任务。

## 插件使用

### Templater

- 新建模板时通过适度弹窗选择学科、章节、状态、评分、来源。
- 题目 / 错题卡可按父文件夹自动加前缀，并生成创建日期、封面字段。
- 不把所有内容都自动化，正文保留人工判断空间。

### Dataview

- `思维导图`、`同类题目汇总`、`章节复习仪表盘` 用普通 DQL 动态查询。
- 推荐查询字段：`type`、`tags`、`method`、`status`、`rating`、`source`、`cover`。
- 不使用 JS 查询，只保留普通 DQL，降低维护成本。

### Callout Manager / Obsidian Callouts

- 只使用默认 callout 类型：`note`、`tip`、`warning`、`example`、`question`、`quote`。
- `warning` 放易错点和待确认；`tip` 放解题入口；`example` 放题目；`quote` 放来源。

### Mind Map NextGen

- `思维导图` 模板的标题层级就是脑图结构。
- 建议顺序：知识体系 -> 方法与题型 -> 题目与错题 -> 复习入口 -> 待确认。

### Paste Image Rename / Mousewheel Image Zoom

- PDF 截图、题图、图像公式粘贴后统一命名。
- 对题图、函数图像、教材截图先放大检查，再写入笔记结论。

### Excalidraw

- 几何图形、积分区域、线代结构图、概率分布关系图，优先做成可复习图解。
- 图解卡里保留 `![[Excalidraw/xxx.excalidraw]]` 或导出图嵌入位。

### Floating TOC / Editing Toolbar / Easy Typing

- 长章节笔记保持清晰标题层级，便于 Floating TOC 跳转。
- Editing Toolbar 适合快速补 callout、加粗、表格。
- Easy Typing 只作为中文输入辅助，不改变模板语法。

### Obsidian Git / OZ Clear Unused Images

- 大批量入库前后先检查 Git 状态，再考虑备份提交。
- OZ Clear Unused Images 只在确认无误后使用；不要在入库过程中提前清理图片。

## 模板目录提示

当前仓库存在 `模版文件夹`，但 `.obsidian/templates.json` 可能仍指向旧目录名 `1.模板`。本次不修改 `.obsidian/` 配置；如果在 Obsidian 里无法从模板命令看到这些文件，请手动检查模板插件目录设置。

## 保护规则

- 不使用 408 笔记体系。
- 不覆盖已有“我的补充”“错题回流”“手写批注”。
- 不确定、OCR 不清、资料缺页时标记 `待确认`，不要编造。
- 新资料和旧笔记冲突时，单独列出冲突点，不直接覆盖。
- 只在明确需要时新增图解、题目、复盘；不要为了形式拆出低价值卡片。
